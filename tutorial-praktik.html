üìã Apa yang Akan Kamu Buat?

Dalam tutorial ini, kamu akan membuat project klasifikasi data penjualan menggunakan algoritma Decision Tree (C4.5 dan CART).

Klasifikasi adalah proses mengelompokkan data ke dalam kategori. Contoh: Email ‚Üí Spam atau Bukan Spam. Dalam project ini: Penjualan ‚Üí Low, Medium, atau High.

üéØ Tujuan Project:

  • Membaca data penjualan e-commerce (1000 transaksi)
  • Membuat kategori penjualan (Low/Medium/High)
  • Melatih komputer untuk memprediksi kategori penjualan
  • Membandingkan 2 metode pembelajaran (C4.5 vs CART)

üìù Langkah 1-4: Menyiapkan Data

1
Buka RapidMiner dan Buat Project Baru
  1. Buka aplikasi RapidMiner Studio
  2. Klik File ‚Üí New Process
  3. Sekarang kamu akan melihat canvas kosong di tengah layar
Process adalah "resep" atau alur kerja yang akan kita buat. Sama seperti membuat resep masakan baru, kita mulai dari halaman kosong.
Simpan project-mu sejak awal! Klik File ‚Üí Save Process As dan beri nama yang jelas, misalnya "Klasifikasi_Penjualan".
‚ùå RapidMiner tidak mau buka / crash:
‚úÖ Solusi: Pastikan Java sudah terinstall. Download dari java.com jika belum ada.
‚ùå Muncul "License expired":
‚úÖ Solusi: Daftar akun gratis di rapidminer.com dan login.
2
Tambahkan Operator "Read CSV"
  1. Di panel Operators (kiri bawah), ketik Read CSV di kotak pencarian
  2. Drag (seret) operator Read CSV ke area canvas (tengah)
  3. Klik operator tersebut, lalu di panel Parameters (kanan):
  4. Klik tombol folder di sebelah csv file
  5. Pilih file Ecommerce_Sales_Prediction_Dataset.csv
  6. Klik Import Configuration Wizard untuk mengatur import
  7. Ikuti wizard sampai selesai (biasanya klik Next terus sampai Finish)

Contoh tampilan Read CSV

Read CSV adalah pintu masuk data ke RapidMiner. CSV (Comma Separated Values) adalah format file yang umum untuk menyimpan data tabel. Tanpa langkah ini, RapidMiner tidak punya data untuk diproses.
Data mentah adalah data yang belum diolah. Dalam file CSV ini, ada 1000+ baris data transaksi penjualan dengan kolom: Date, Product_Category, Price, Discount, Customer_Segment, Marketing_Spend, dan Units_Sold.
Pastikan lokasi file CSV sudah benar! Jika salah, RapidMiner akan error saat dijalankan.
‚ùå "Could not read file" / "File not found":
‚úÖ Solusi: Path file salah. Klik folder lagi, pilih ulang file CSV dengan benar.
‚ùå Data kosong / tidak muncul:
‚úÖ Solusi: Klik "Import Configuration Wizard", centang "first row as names", klik Next sampai Finish.
‚ùå Kolom berantakan:
‚úÖ Solusi: Di wizard, ubah "column separator" ke semicolon (;) atau comma (,).
3
Tambahkan Operator "Select Attributes"
  1. Cari operator Select Attributes di panel Operators
  2. Drag ke canvas, letakkan di sebelah kanan Read CSV
  3. Hubungkan output Read CSV ke input Select Attributes:
    • Klik titik kecil di sisi kanan Read CSV
    • Tarik garis ke titik di sisi kiri Select Attributes
  4. Klik Select Attributes, di Parameters pilih:
    • attribute filter type: subset
    • Klik Select Attributes dan centang semua kolom yang mau dipakai

Contoh tampilan Select Attributes

Tidak semua kolom data selalu berguna. Operator ini membantumu memilih kolom yang relevan saja. Misalnya, kalau ada kolom "ID Transaksi", itu tidak berguna untuk prediksi dan bisa dibuang.
Dalam project ini, kita pakai semua kolom. Tapi di project nyata, kamu mungkin perlu membuang kolom yang tidak relevan untuk hasil yang lebih baik.
‚ùå "No attributes selected":
‚úÖ Solusi: Kamu belum memilih kolom. Klik "Select Attributes" dan centang kolom yang dibutuhkan.
‚ùå Garis tidak bisa ditarik:
‚úÖ Solusi: Klik tepat di titik kecil (port) di sisi operator. Coba zoom in jika susah.
4
Tambahkan Operator "Generate Attributes"
  1. Cari dan drag operator Generate Attributes
  2. Hubungkan dari Select Attributes
  3. Klik operator ini, lalu di Parameters:
  4. Klik Edit List di sebelah "function descriptions"
  5. Isi seperti ini:
    • Attribute name: status_penjualan
    • Function expressions:
      if(Units_Sold <= 20, "Low", if(Units_Sold <= 51, "Medium", "High"))
  6. Klik OK

Contoh tampilan Generate Attributes

Kita sedang membuat label/kategori untuk data. Data asli hanya punya angka (Units_Sold = 32), tapi komputer butuh kategori (Low/Medium/High) untuk belajar klasifikasi. Ini seperti guru yang memberi nilai huruf berdasarkan skor angka.
Labeling/Pelabelan adalah proses mengubah data numerik menjadi kategori. Ini penting karena algoritma klasifikasi membutuhkan kategori yang jelas untuk dipelajari.

Aturan yang kita buat:
‚Ä¢ Units_Sold ‚â§ 20 ‚Üí "Low" (penjualan rendah)
‚Ä¢ Units_Sold 21-51 ‚Üí "Medium" (penjualan sedang)
‚Ä¢ Units_Sold > 51 ‚Üí "High" (penjualan tinggi)
Hati-hati mengetik rumus! Satu huruf salah bisa menyebabkan error. Pastikan tanda kurung dan tanda kutip sudah benar.
‚ùå "Unknown attribute Units_Sold":
‚úÖ Solusi: Nama kolom salah. Pastikan persis: Units_Sold (pakai underscore, S besar).
‚ùå "Expression parse error" / "Syntax error":
‚úÖ Solusi: Rumus salah. Copy ini:
if(Units_Sold <= 20, "Low", if(Units_Sold <= 51, "Medium", "High"))
‚ùå Tanda kutip error:
‚úÖ Solusi: Pakai kutip lurus " bukan kutip melengkung ". Ketik manual, jangan copy dari Word.

üìù Langkah 5-8: Menyiapkan Model

5
Tambahkan Operator "Filter Examples"
  1. Cari dan drag operator Filter Examples
  2. Hubungkan dari Generate Attributes
  3. Di Parameters:
    • condition class: custom_filters
    • Klik Add Filters
    • Pilih: Units_Sold, gt (greater than), 51
    • Centang invert filter = Yes

Contoh tampilan Filter Examples

Dalam project ini, kita hanya fokus ke 2 kategori: Low dan Medium. Data dengan kategori High dibuang supaya model lebih sederhana dan fokus. Ini seperti menyederhanakan soal ujian.
Filtering adalah proses menyaring data. "gt" artinya "greater than" (lebih besar dari). Dengan "invert filter", kita membalik logika: yang tersisa adalah data yang TIDAK lebih besar dari 51.
Di project nyata, mungkin kamu tidak perlu filter. Tapi kadang data outlier (nilai ekstrem) perlu dihapus agar model lebih akurat.
‚ùå Semua data hilang setelah filter:
‚úÖ Solusi: Cek apakah "invert filter" dicentang. Kalau salah, data yang seharusnya dibuang malah dipertahankan.
‚ùå Filter tidak berfungsi:
‚úÖ Solusi: Pastikan condition class = "custom_filters" dan nama atribut sesuai (Units_Sold).
6
Tambahkan Operator "Set Role"
  1. Cari dan drag operator Set Role
  2. Hubungkan dari Filter Examples
  3. Di Parameters:
    • attribute name: status_penjualan
    • target role: label

Contoh tampilan Set Role

Ini langkah SANGAT PENTING! Kita memberitahu RapidMiner: "Kolom status_penjualan adalah TARGET yang mau kita prediksi." Tanpa ini, komputer tidak tahu apa yang harus dipelajari.
Dalam machine learning, ada 2 jenis kolom:
‚Ä¢ Label/Target: Kolom yang mau diprediksi (status_penjualan)
‚Ä¢ Features/Fitur: Kolom lain yang jadi petunjuk (Price, Discount, dll)

Komputer akan belajar pola dari fitur untuk menebak label.
‚ùå "Attribute not found":
‚úÖ Solusi: Kolom status_penjualan belum ada. Pastikan langkah 4 (Generate Attributes) sudah benar.
‚ùå Decision Tree error "No label defined":
‚úÖ Solusi: Set Role belum diatur. Pastikan target role = "label" sudah diisi.
7
Tambahkan Operator "Multiply"
  1. Cari dan drag operator Multiply
  2. Hubungkan dari Set Role
  3. Tidak perlu atur parameter apapun

Contoh tampilan Multiply

Kita mau membandingkan 2 algoritma (C4.5 dan CART). Data perlu di-copy supaya bisa masuk ke 2 proses berbeda secara paralel. Ini seperti fotocopy soal ujian untuk 2 siswa.
Operator Multiply akan punya beberapa output (out 1, out 2, dst). Nanti kita hubungkan masing-masing ke proses validasi yang berbeda.
8
Tambahkan 2 Operator "Validation" (X-Validation)
  1. Cari operator Validation atau X-Validation
  2. Drag 2 kali ke canvas (beri nama "Validation" dan "Validation (2)")
  3. Hubungkan:
    • Output 1 Multiply ‚Üí Input Validation (untuk C4.5)
    • Output 2 Multiply ‚Üí Input Validation (2) (untuk CART)
  4. Untuk kedua Validation, atur Parameters:
    • number of validations: 10
    • sampling type: automatic

Contoh tampilan X-Validation

Cross Validation adalah cara menguji model yang adil. Data dibagi jadi 10 bagian. Komputer belajar dari 9 bagian, diuji dengan 1 bagian. Diulang 10 kali dengan bagian berbeda yang jadi ujian. Hasil akhir adalah rata-rata dari 10 pengujian.
Kenapa perlu validasi?
Kalau kita langsung test pakai data yang sama dengan data latihan, hasilnya bisa "curang" - komputer cuma hafal, bukan benar-benar paham pola. Cross validation memastikan model benar-benar bisa menebak data BARU yang belum pernah dilihat.
‚ùå "Input is not connected":
‚úÖ Solusi: Output Multiply belum dihubungkan ke Validation. Tarik garis dari output Multiply ke input Validation.
‚ùå Validation kosong / tidak ada isi:
‚úÖ Solusi: Double-click Validation untuk masuk ke dalamnya. Di dalam ada 2 area yang perlu diisi.

üìù Langkah 9-12: Membuat Model Decision Tree

9
Masuk ke Dalam Validation dan Tambahkan Decision Tree C4.5
  1. Double-click pada kotak Validation pertama untuk masuk ke dalamnya
  2. Kamu akan lihat ada 2 area: Training dan Testing
  3. Di area Training:
    • Cari operator Decision Tree
    • Drag ke dalam area Training
    • Hubungkan port "training" ke input Decision Tree
    • Hubungkan output Decision Tree ke port "model"
  4. Klik Decision Tree, atur Parameters:
    • criterion: gain_ratio (ini yang membuat jadi C4.5)
    • maximal depth: 20
    • apply pruning: true
Decision Tree adalah algoritma yang membuat aturan berbentuk pohon. Seperti flowchart yang bertanya: "Kalau harganya segini, terus diskonnya segitu, maka penjualannya..."

Gain Ratio adalah kriteria yang dipakai C4.5 untuk memilih pertanyaan terbaik di setiap cabang pohon.
Pruning (Pemangkasan) adalah proses memotong cabang pohon yang tidak terlalu penting. Ini mencegah model jadi terlalu kompleks dan "hafalan". Model yang terlalu hafalan disebut overfitting - bagus di data latihan tapi jelek di data baru.
‚ùå "No training data":
‚úÖ Solusi: Port "training" di dalam Validation belum dihubungkan ke Decision Tree.
‚ùå Decision Tree tidak muncul di Operators:
‚úÖ Solusi: Ketik "Decision Tree" di search box. Kalau tetap tidak ada, coba ketik "Tree".
‚ùå Tidak bisa keluar dari dalam Validation:
‚úÖ Solusi: Klik ikon panah di pojok kiri atas canvas untuk kembali ke main process.
10
Di Area Testing: Tambahkan Apply Model dan Performance
  1. Masih di dalam Validation, pindah ke area Testing
  2. Drag operator Apply Model
  3. Drag operator Performance (Classification)
  4. Hubungkan:
    • Port "model" ‚Üí Apply Model (input atas)
    • Port "test set" ‚Üí Apply Model (input bawah)
    • Output Apply Model ‚Üí input Performance
    • Output Performance ‚Üí port "averagable 1"
  5. Klik Performance, pastikan accuracy dicentang
Apply Model: Menerapkan model yang sudah dilatih ke data test. Seperti siswa mengerjakan soal ujian setelah belajar.

Performance: Menghitung seberapa akurat prediksinya. Berapa persen yang ditebak dengan benar.
Setelah selesai mengatur di dalam Validation, klik tombol panah di pojok kiri atas untuk kembali ke tampilan utama (main process).
11
Ulangi untuk Validation (2) dengan CART
  1. Double-click pada Validation (2)
  2. Ulangi langkah 9-10 dengan perbedaan:
  3. Saat mengatur Decision Tree, pilih:
    • criterion: gini_index (ini yang membuat jadi CART)
  4. Parameter lainnya sama saja
Sekarang kita punya 2 model dengan metode berbeda:
‚Ä¢ C4.5 menggunakan Gain Ratio
‚Ä¢ CART menggunakan Gini Index

Ini memungkinkan kita membandingkan mana yang lebih baik untuk dataset ini.
Gini Index adalah cara lain untuk memilih pertanyaan terbaik di pohon keputusan. Konsepnya: Ukur seberapa "campur aduk" data di setiap cabang. Cabang yang lebih "murni" (isinya satu kategori) lebih bagus.
12
Hubungkan Output ke Result
  1. Kembali ke tampilan utama (main process)
  2. Hubungkan output dari kedua Validation ke port result di kanan:
    • Validation output "model" ‚Üí res (result)
    • Validation output "averagable" ‚Üí res
    • Validation (2) output "model" ‚Üí res
    • Validation (2) output "averagable" ‚Üí res

Tampilan keseluruhan proses yang sudah lengkap

Port "result" adalah tempat output yang mau kita lihat di tab Results. Kalau tidak dihubungkan, kita tidak bisa melihat hasilnya!

üìä Memahami Hasil

Setelah proses selesai, kamu akan melihat beberapa tab hasil:

1. Decision Tree (Pohon Keputusan)

Hasil Decision Tree

Ini adalah "otak" model yang sudah belajar. Baca dari atas ke bawah:
‚Ä¢ Akar pohon menunjukkan pertanyaan pertama
‚Ä¢ Cabang menunjukkan jawaban
‚Ä¢ Daun (ujung) menunjukkan prediksi final

Contoh dalam project ini:
"Kalau Units_Sold ‚â§ 20.5, maka prediksi = Low"
"Kalau Units_Sold > 20.5, maka prediksi = Medium"

2. Performance / Confusion Matrix

Confusion Matrix dan Akurasi

Accuracy (Akurasi): Persentase prediksi yang benar. 100% berarti semua tebakan benar!

Confusion Matrix: Tabel yang menunjukkan:
Ditebak Medium Ditebak Low
Sebenarnya Medium 889 ‚úì (benar) 0
Sebenarnya Low 0 109 ‚úì (benar)
Angka diagonal (889 dan 109) adalah yang ditebak dengan benar!
Kenapa akurasinya 100%?
Karena kolom status_penjualan dibuat langsung dari Units_Sold dengan aturan yang jelas. Jadi Decision Tree dengan mudah menemukan pola tersebut. Di dataset nyata, akurasi biasanya lebih rendah karena polanya lebih kompleks.
