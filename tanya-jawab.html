5
Kenapa memilih algoritma Decision Tree?
‚úÖ Jawaban:

Decision Tree dipilih karena beberapa alasan:

  1. Mudah dipahami - Hasilnya berupa aturan yang bisa dibaca manusia
  2. Tidak perlu normalisasi data - Bisa langsung memproses data mentah
  3. Bisa menangani data kategorikal dan numerik
  4. Visualisasi jelas - Bisa dilihat dalam bentuk pohon
  5. Cocok untuk klasifikasi - Sangat baik untuk mengelompokkan data
Kalau ditanya lebih lanjut, bandingkan dengan algoritma lain: "Decision Tree lebih mudah diinterpretasi dibanding Neural Network atau SVM."

Apa bedanya C4.5 dan CART?
‚úÖ Jawaban:
Aspek C4.5 CART
Kriteria Split Gain Ratio Gini Index
Tipe Pohon Bisa multi-way (banyak cabang) Binary (selalu 2 cabang)
Missing Values Bisa menangani langsung Perlu handling khusus
Output Aturan/rules Pohon biner
Gain Ratio (C4.5) = Mengukur seberapa banyak informasi yang didapat dari suatu atribut, dinormalisasi dengan split information.

Gini Index (CART) = Mengukur tingkat ketidakmurnian. Nilai 0 berarti semua data di node itu sama kelasnya (murni).

Apa itu Cross Validation? Kenapa pakai 10-fold?
‚úÖ Jawaban:

Cross Validation adalah teknik untuk menguji model dengan cara:

  1. Data dibagi menjadi K bagian (dalam kasus ini K=10)
  2. Secara bergantian, 1 bagian jadi data test, sisanya jadi data training
  3. Proses diulang K kali
  4. Hasil akhir adalah rata-rata dari semua pengujian

Kenapa 10-fold?

  • 10 adalah standar industri yang sudah terbukti efektif
  • Cukup banyak untuk validasi yang akurat
  • Tidak terlalu banyak sehingga proses masih cepat
"Cross validation itu seperti ujian 10 kali dengan soal berbeda, Pak/Bu. Nilai akhirnya dirata-rata supaya hasilnya lebih adil dan tidak kebetulan."

  2. Data tidak seimbang - Mungkin data "High" terlalu sedikit sehingga model sulit mempelajarinya
  3. Scope project - Project ini fokus membedakan penjualan rendah dan sedang
Kalau tidak yakin alasannya, jawab dengan jujur: "Dalam project ini, kami fokus pada klasifikasi 2 kelas untuk menyederhanakan analisis. Di project yang lebih kompleks, kategori High bisa dimasukkan."
9
Apa fungsi operator Set Role?
‚úÖ Jawaban:

Set Role digunakan untuk menentukan peran setiap kolom dalam proses machine learning:

  • Label = Kolom yang mau diprediksi (target variabel)
  • ID = Kolom identifikasi unik
  • Weight = Bobot data
  • Regular/Attribute = Kolom fitur untuk prediksi (default)

Dalam project ini, status_penjualan di-set sebagai label karena itulah yang mau kita prediksi.

Tanpa Set Role, RapidMiner tidak tahu kolom mana yang harus diprediksi!
10
Apa itu Confusion Matrix?
‚úÖ Jawaban:

Confusion Matrix adalah tabel yang menunjukkan perbandingan antara prediksi model dengan nilai sebenarnya:

Diprediksi: Medium Diprediksi: Low
Aktual: Medium True Positive (TP) False Negative (FN)
Aktual: Low False Positive (FP) True Negative (TN)

Cara baca:

  • Diagonal (hijau) = Prediksi BENAR
  • Non-diagonal (merah) = Prediksi SALAH

Dalam project ini: 889 Medium ditebak benar, 109 Low ditebak benar, 0 salah = 100% akurat!

11
Apa itu Pruning? Kenapa diaktifkan?
‚úÖ Jawaban:

Pruning (Pemangkasan) adalah proses memotong cabang pohon yang tidak signifikan.

Tujuan Pruning:

  • Mencegah overfitting (model terlalu hafalan)
  • Membuat pohon lebih sederhana dan mudah dipahami
  • Meningkatkan generalisasi ke data baru
"Pruning itu seperti pangkas rumput, Pak/Bu. Kalau pohon keputusan terlalu rimbun (kompleks), kita potong cabang yang tidak penting supaya lebih rapi dan efektif."

Kenapa akurasinya bisa 100%? Apakah itu bagus?
‚úÖ Jawaban:

Akurasi 100% tercapai karena:

  • Kolom status_penjualan dibuat langsung dari Units_Sold dengan aturan yang jelas
  • Polanya sangat sederhana: ‚â§20 = Low, >20 = Medium
  • Decision Tree dengan mudah menemukan pola ini
Apakah ini bagus?
Untuk tujuan pembelajaran: YA, ini menunjukkan algoritma bekerja dengan benar.

Di dunia nyata: Akurasi 100% bisa jadi tanda data leakage atau overfitting. Biasanya akurasi realistis adalah 70-95%.
Jawab dengan jujur bahwa ini karena pola yang sederhana. Dosen akan menghargai kejujuran dan pemahaman mendalam.

Apa itu Overfitting dan bagaimana mencegahnya?
‚úÖ Jawaban:

Overfitting adalah kondisi di mana model terlalu "hafal" data training sehingga:

  • Performa di data training: SANGAT BAGUS
  • Performa di data baru: JELEK

Analogi: Seperti siswa yang hanya hafal soal latihan persis, tapi tidak paham konsepnya. Saat soal ujian berbeda sedikit, dia tidak bisa menjawab.

Cara mencegah:

  1. Pruning - Memangkas pohon yang terlalu kompleks
  2. Cross Validation - Menguji dengan data yang berbeda-beda
  3. Batasi kedalaman pohon - Parameter maximal_depth
  4. Minimal leaf size - Minimal data di setiap daun

üìö Konsep Pendukung yang Perlu Dipahami

1. Jenis-jenis Machine Learning

Jenis Penjelasan Contoh
Supervised Learning Belajar dari data yang sudah ada labelnya Klasifikasi, Regresi
Unsupervised Learning Mencari pola tanpa label Clustering, Dimensionality Reduction
Reinforcement Learning Belajar dari reward/punishment Game AI, Robot
Project ini adalah Supervised Learning karena kita punya label (status_penjualan) yang sudah ditentukan.

2. Klasifikasi vs Regresi

Klasifikasi Regresi
Memprediksi kategori/kelas Memprediksi nilai numerik
Contoh: Low/Medium/High Contoh: Harga rumah = Rp 500 juta
Output: Diskrit/terbatas Output: Kontinu/tidak terbatas

3. Metrik Evaluasi untuk Klasifikasi

Metrik Rumus Penjelasan
Accuracy (TP+TN) / Total Persentase prediksi yang benar secara keseluruhan
Precision TP / (TP+FP) Dari yang diprediksi positif, berapa yang benar-benar positif
Recall TP / (TP+FN) Dari yang sebenarnya positif, berapa yang berhasil diprediksi
F1-Score 2 √ó (P√óR)/(P+R) Rata-rata harmonis precision dan recall

4. Tahapan Data Mining (CRISP-DM)

  1. Business Understanding - Pahami masalah bisnis
  2. Data Understanding - Pahami data yang ada

  4. Modeling - Buat dan latih model
